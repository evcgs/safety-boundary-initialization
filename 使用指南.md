## 一、安装与配置

### 1.1 安装步骤

#### 方式一：复制到 skills 目录

```bash
# 1. 下载或克隆这个项目
git clone https://github.com/evcgs/safety-boundary-initialization.git

# 2. 复制到 OpenClaw skills 目录
cp -r safety-boundary-initialization ~/.openclaw/skills/
```

1. 创建目录：`mkdir -p ~/.openclaw/skills/safety-boundary-initialization`
2. 将所有文件复制到该目录

### 1.2 验证安装

检查文件是否正确放置：

## 二、第一次对话流程

### 2.1 技能触发

当用户第一次与 AI 助手对话时，技能会自动触发。

### 2.2 引导步骤

#### 步骤 1：自我介绍

AI 助手会说：

```
你好！我是你的 AI 助手。在开始协作之前，我们需要先建立一些基本的规则和边界，这样可以让我们的协作更高效、更信任。

我会引导你完成以下几个步骤：
1. 确认授权边界 - 明确什么能做、什么不能做
2. 了解结果确认模板 - 如何明确任务目标
3. 建立沟通反馈机制 - 什么时候该问、什么时候该等

准备好了吗？我们开始吧！
```

#### 步骤 2：确认授权边界

AI 助手会展示授权边界清单，并询问：

```
这是预设的授权边界清单。你可以：
1. 直接使用预设内容
2. 修改某些条款
3. 重新制定

你想怎么做？
```

**用户操作**：
- 阅读授权边界清单
- 根据需要修改
- 确认最终版本

#### 步骤 3：了解结果确认模板

AI 助手会展示结果确认模板，并说明：

```
这是结果确认模板。每次正式任务开始前，我们会用这个模板来确认：
- 核心任务目标（可量化、可验证）
- 安全护栏（不可突破的约束）
- 验收标准（什么算完成）
- 时间要求
- 风险等级与授权

你可以先看看，有什么问题吗？
```

**用户操作**：
- 阅读结果确认模板
- 理解填写方法
- 确认已知晓如何使用

#### 步骤 4：建立沟通反馈机制

AI 助手会展示沟通反馈机制，并确认：

```
这是沟通反馈机制。我们约定：
- 什么情况下我会打断你
- 什么情况下我不会打扰你
- 进度汇报的频率和方式
- 特殊场景的处理方式

你觉得这样可以吗？有什么需要调整的吗？
```

**用户操作**：
- 阅读沟通反馈机制
- 根据需要调整
- 确认最终版本

#### 步骤 5：完成确认

AI 助手会总结并确认：

```
太好了！我们已经完成了所有设置：
✅ 授权边界已确认
✅ 结果确认模板已了解
✅ 沟通反馈机制已建立

现在我们可以开始协作了！有什么需要我帮忙的吗？
```

AI 助手会按照沟通反馈机制执行：

- **正常执行中**：不打扰用户
- **遇到问题**：主动打断，说明问题
- **需要决策**：列出选项，给出建议
- **进度汇报**：按约定频率同步进度

### 3.3 任务完成后：验收和总结

AI 助手会提交交付物并总结：

```

- 信息收集阶段可以更高效
```

## 四、自定义配置

### 4.1 修改授权边界

编辑 `授权边界清单.md`，根据你的需求调整：

- 添加新的操作类型
- 调整风险等级
- 修改授权方式
- 补充禁止操作

### 4.2 调整结果确认模板

编辑 `结果确认模板.md`：

- 修改默认安全护栏
- 添加新的确认项
- 调整模板结构
- 补充更多示例

### 4.3 优化沟通反馈机制

编辑 `沟通反馈机制.md`：

- 调整打断场景
- 修改汇报频率
- 补充特殊场景
- 调整沟通渠道

## 五、最佳实践

### 5.1 授权边界

1. **先紧后松**：刚开始时边界可以严一些，熟悉后再放宽
2. **定期回顾**：每季度回顾一次，调整不合适的地方
3. **记录例外**：如果有特殊情况，记录下来，作为下次调整的依据

### 5.2 结果导向

1. **越具体越好**：目标描述要具体、可量化
2. **验收标准明确**：什么算完成，要清清楚楚
3. **安全护栏清晰**：什么绝对不能做，要明明白白

### 5.3 沟通反馈

1. **信任 AI**：给 AI 足够的自主空间
2. **及时反馈**：如果觉得有问题，及时提出来
3. **定期复盘**：每两周回顾一次沟通机制，调整不合适的地方

## 六、常见问题

### 6.1 技能没有自动触发怎么办？

**答**：手动触发技能，或者直接开始设置流程。

### 6.2 我想修改授权边界怎么办？

**答**：直接编辑 `授权边界清单.md` 文件，然后告诉 AI 助手更新后的内容。

### 6.3 任务执行中遇到问题怎么办？

**答**：AI 助手会主动打断你，说明问题并寻求指示。你也可以随时问 AI 助手：「现在进展如何？」

### 6.4 我觉得汇报频率不合适怎么办？

**答**：直接告诉 AI 助手你想要的汇报频率，或者编辑 `沟通反馈机制.md` 文件。

### 6.5 这套机制太复杂了，能不能简化？

**答**：可以！你可以：
1. 只使用授权边界清单
2. 跳过结果确认模板（简单任务不用）
3. 简化沟通反馈机制

根据你的需求定制就好。

---

## 七、总结

使用这个技能的核心是：

1. **先建立规则**：在开始协作前，先把边界、目标、沟通机制说清楚
2. **在实践中迭代**：先用起来，再根据实际情况调整
3. **保持开放沟通**：有什么问题及时提出来，一起优化

希望这个技能能帮助你建立高效、信任的 AI 协作关系！
