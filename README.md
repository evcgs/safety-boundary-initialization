# Safety Boundary Initialization

一个在第一次对话时触发的技能，帮助用户建立安全边界、授权级别和沟通协议。

---

## 简介

这个技能基于「风险可控内授予最高权限」、「结果导向」、「不干涉过程」三大原则，帮助用户在第一次使用 AI 助手时，快速建立清晰的安全边界和协作机制。

---

## 快速开始

### 1. 引入技能

将这个技能复制到你的 OpenClaw skills 目录中：

```bash
cp -r safety-boundary-initialization ~/.openclaw/skills/
```

### 2. 第一次对话

当用户第一次与 AI 助手对话时，技能会自动触发，引导用户完成设置。

## 核心文件

### 📄 授权边界清单.md

定义了不同风险等级的操作类型和授权方式：

- **低风险**：完全自主执行，无需确认
- **中风险**：自主执行，事后报备
- **高风险**：事前审批，确认后执行
- **永远禁止**：绝对不能做的操作

### 📄 结果确认模板.md

每次任务开始前使用的确认模板：

- 核心任务目标（可量化、可验证）
- 安全护栏目标（不可突破的约束）
- 验收标准（什么算完成）
- 时间要求
- 风险等级与授权方式

### 📄 沟通反馈机制.md

明确什么时候打断、什么时候不干预：

- 什么情况下必须打断
- 什么情况下完全不干预
- 进度汇报约定
- 特殊场景处理

## 三大原则

### 1. 风险可控内授予最高权限

先锁死风险边界，再给 AI 最大的操作空间。

```
低风险 → 自主决策
中风险 → 先做后说
高风险 → 先说后做
```

### 2. 结果导向

人类只定义可量化、可验证的最终目标，AI 自主决定如何实现目标。

```
❌ 不好："写一篇好文章"
✅ 好："写一篇3000字的AI Agent行业分析报告，包含5个标杆案例"
```

### 3. 不干涉过程

「不干涉」为常态，「打断」为例外，「有问题再叫我」模式。

```
正常执行 → 不干预
遇到问题 → 主动打断
需要决策 → 列出选项
```

## 创建背景

这个技能源于真实的 AI 协作实践。在与 AI 助手的长期协作中，我们发现：

1. **模糊的边界导致效率低下**：用户不知道 AI 能做什么、不能做什么
2. **缺乏结果导向导致目标偏离**：任务目标不清晰，验收标准不明确
3. **沟通不畅导致信任问题**：不知道什么时候该问、什么时候该等

通过建立清晰的边界、结果导向的目标设定、有效的沟通机制，我们可以大幅提升 AI 协作的效率和信任度。
